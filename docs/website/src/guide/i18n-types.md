## 方案背景

在 **typescript** 项目中使用 **I18n** 实现国际化时，传统方式存在以下痛点：

1. 翻译 key 手写易出错，无类型提示和校验；
2. 翻译命名空间（namespace）和语言类型管理混乱；
3. `$t` 函数调用时参数不规范，运行时才暴露错误；
4. 翻译文件变更后，类型无法自动同步，需手动维护。

本方案通过自定义 Vite 插件 + 类型生成脚本，实现「翻译文件 → 自动生成类型 → 开发阶段实时校验」的全链路类型安全，彻底解决上述问题。

## 核心目标

1. 自动扫描所有国际化 JSON 文件，提取翻译 key 和命名空间；
2. 生成包含翻译 key、命名空间、语言类型的 TypeScript 类型文件；
3. 结合 Vite 热更新，翻译文件变更时自动更新类型；
4. 为 `$t` 函数添加严格类型约束，实现编辑器自动提示和编译期校验。

## 实现原理

### 核心流程

1. 扫描 `locales/langs` 目录下的多语言 JSON 文件，按命名空间分类收集所有翻译 key；
2. 基于收集到的 key 和命名空间，生成结构化的 TypeScript 类型声明；
3. 封装 Vite 插件，在开发阶段监听翻译文件变更，实时更新类型文件；
4. 扩展 Vue I18n 的 `$t` 函数类型，绑定生成的翻译 key 类型，实现类型校验。

### 关键技术点

- 文件系统遍历：通过 fs 模块递归扫描 locales 目录，识别语言目录和 JSON 翻译文件；
- 哈希校验：计算翻译文件内容哈希值，避免热更新时重复生成类型；
- 类型声明生成：动态拼接 TypeScript 类型代码，生成包含命名空间、翻译 key、语言类型的类型文件；
- Vite 热更新监听：通过 handleHotUpdate 钩子监听 JSON 文件变更，触发类型重新生成。

## 实现步骤

### 编写类型生成核心函数

实现 `generateI18nTypes` 函数，核心逻辑包括：

#### 步骤1：扫描目录与收集文件

- 读取 langs 目录下的所有语言子目录（如 zh-CN、en-US）；
- 遍历每个语言目录下的 JSON 文件，按文件名提取命名空间（如 auth.json → auth）。

#### 步骤2：递归收集翻译 key

定义 `collectKeys` 递归函数，解析 JSON 文件的嵌套结构，生成完整的翻译 key（如 "login.form.username"）：

- 遍历 JSON 对象的每个键值对，拼接前缀生成完整 key；
- 若值为嵌套对象，继续递归解析；若为基础类型（字符串/数字），则收集该 key。

#### 步骤3：生成类型文件内容

动态拼接 TypeScript 类型代码，核心生成以下类型：

1. I18nKeys 命名空间：按模块拆分翻译 key 类型（如 I18nKeys.auth 包含 auth 模块所有 key）；
2. I18nFullKey：组合命名空间和 key 的完整翻译键类型（如 `auth.login.title`）；
3. I18n 命名空间：包含语言类型（LangType）、语言选项（LangOption）、`$t` 函数类型；
4. `$t` 函数重载：覆盖 Vue I18n 原生 `$t` 函数的所有调用方式，绑定 I18nFullKey 类型。

### 封装 Vite 插件

实现 `generateI18nTypesPlugin` 插件，集成到 Vite 构建流程：

#### 步骤1：构建启动时生成类型

在 `buildStart` 钩子中，仅开发环境执行：

- 计算翻译文件的初始哈希值，避免重复生成；
- 调用 `generateI18nTypes` 生成初始类型文件。

#### 步骤2：热更新监听与类型更新

在 `handleHotUpdate` 钩子中：

- 过滤仅监听 locales 目录下的 JSON 文件变更；
- 重新计算文件哈希值，仅当内容变化时触发类型重新生成；
- 输出变更日志，便于开发调试。

## 方案优势

### 开发体验提升

- 编辑器自动提示：输入 `$t` 时，自动补全所有合法的翻译 key；
- 编译期错误校验：翻译 key 拼写错误、命名空间错误等问题在开发阶段暴露，避免运行时错误；
- 热更新自动同步：修改翻译 JSON 文件后，类型文件实时更新，无需重启开发服务。

### 可维护性提升

- 类型文件自动生成：无需手动维护类型，翻译文件变更后类型自动同步；
- 结构化管理：按业务模块拆分翻译文件，命名空间类型清晰；
- 语言类型约束：限制语言切换的取值范围，避免非法语言类型。

### 扩展性强

- 支持多语言扩展：新增语言目录（如 ja、ko）后，类型自动识别；
- 支持嵌套 key：兼容任意层级的翻译 key 嵌套结构；
- 兼容 Vue I18n 所有 `$t` 函数调用方式：覆盖复数、默认值、命名参数等场景。

## 常见问题与解决方案

### 类型文件生成失败

- 检查目录路径：确保 localesDir 和 outFile 路径解析正确（使用 path.resolve 避免相对路径问题）；
- 检查 JSON 文件格式：确保翻译文件是合法的 JSON 格式，无语法错误；
- 权限问题：确保脚本有读写 typings 目录的权限。

### 热更新时类型未更新

- 检查文件监听范围：确保 handleHotUpdate 钩子过滤了正确的文件路径（localesDir 下的 JSON 文件）；
- 哈希值计算错误：确认 calculateHash 函数正确读取所有 JSON 文件内容，避免漏读；
- 环境判断：确保 NODE_ENV 为 development，生产环境不执行热更新逻辑。

### `$t` 函数类型未生效

- 检查 Vue 类型扩展：确保在 Vue 实例类型扩展中正确绑定 `I18n.$T` 类型；
- 关闭 legacy 模式：Vue I18n 需启用 composition API（legacy: false），否则 `$t` 函数类型可能不兼容；
- 类型文件导入：确保 typings 目录在 tsconfig.json 的 include 范围内，TypeScript 能识别生成的类型。

## 待优化项

### 性能优化

- 增量生成：仅更新变更的命名空间类型，而非全量生成；
- 缓存机制：将哈希值缓存到文件，避免每次热更新都读取所有 JSON 文件；

### 功能扩展

- 重复 key 检测：扫描时检测重复的翻译 key，输出警告日志；
- 缺失 key 检测：对比不同语言包的 key，检测缺失的翻译项；
- 集成 CI 校验：在 CI 流程中执行类型生成脚本，确保翻译文件和类型一致。

## 总结

本方案通过「自动扫描 + 类型生成 + Vite 插件监听」的组合，为 Vue I18n 提供了完整的类型安全保障，核心价值体现在：

- 提效：编辑器自动提示，减少手写 key 的错误和查错成本；
- 降错：编译期校验，避免运行时因翻译 key 问题导致的展示异常；
- 易维护：类型自动同步，无需手动维护，适配翻译文件的频繁变更；
- 标准化：规范翻译文件的目录结构和使用方式，提升团队协作效率。

该方案适用于所有使用 Vue I18n 的 Vue 3 项目，尤其适合中大型项目，能显著提升国际化开发的效率和稳定性。
