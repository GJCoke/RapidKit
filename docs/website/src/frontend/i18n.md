# 国际化

::: info
本项目前端基于 [Soybean Admin](https://github.com/soybeanjs/soybean-admin) 开发。基础功能和约定请参考 [Soybean Admin 文档](https://docs.soybeanjs.cn/zh/)。
:::

## 概述

项目使用 Vue I18n 11 实现国际化，采用 Composition API 模式（`legacy: false`），默认语言为 `zh-CN`，回退语言为 `en`。

## 初始化配置

I18n 在 `src/locales/index.ts` 中初始化：

```ts
const i18n = createI18n({
  locale: localStg.get("lang") || "zh-CN",
  fallbackLocale: "en",
  messages,
  legacy: false, // 使用 Composition API 模式
})
```

导出 `$t` 函数供非组件上下文使用（如 Store、路由守卫）：

```ts
export const $t = i18n.global.t as I18n.$T
```

## 语言文件结构

语言文件按 `{locale}/{namespace}.json` 格式组织：

```
src/locales/langs/
  ├── zh-CN/
  │     ├── common.json      # 通用文本
  │     ├── route.json        # 路由/菜单名称
  │     ├── page.json         # 页面文本
  │     ├── form.json         # 表单相关
  │     ├── request.json      # 请求提示
  │     ├── theme.json        # 主题设置
  │     ├── system.json       # 系统管理
  │     ├── datatable.json    # 数据表格
  │     ├── dropdown.json     # 下拉菜单
  │     └── icon.json         # 图标选择器
  └── en-US/
        └── (相同结构)
```

### 动态加载

语言文件通过 Vite 的 `import.meta.glob` 以 eager 模式加载，按文件名自动映射为 namespace：

```ts
const modules = import.meta.glob("./langs/**/*.json", { eager: true })
// 自动解析路径为 messages[locale][namespace] 结构
```

## Namespace 列表

| Namespace   | 说明                           | 使用示例                               |
| ----------- | ------------------------------ | -------------------------------------- |
| `common`    | 通用文本（确认、取消、操作等） | `$t('common.confirm')`                 |
| `route`     | 路由和菜单名称                 | `$t('route.home')`                     |
| `page`      | 页面内容文本                   | `$t('page.login.common.loginSuccess')` |
| `form`      | 表单标签和验证消息             | `$t('form.required')`                  |
| `request`   | 请求错误提示                   | `$t('request.timeout')`                |
| `theme`     | 主题设置界面文本               | `$t('theme.darkMode')`                 |
| `system`    | 系统管理模块文本               | `$t('system.role.title')`              |
| `datatable` | 数据表格操作文本               | `$t('datatable.itemCount')`            |
| `dropdown`  | 下拉菜单选项                   | `$t('dropdown.closeCurrent')`          |
| `icon`      | 图标选择器                     | `$t('icon.label')`                     |

## 类型安全的 I18n Key

项目通过自定义 Vite 插件 `vite-plugin-generate-i18n-types` 自动生成类型定义。

### 工作原理

1. 插件在开发服务器启动时扫描 `src/locales/langs/` 下所有 JSON 文件
2. 递归收集所有翻译 key，按 namespace 分组
3. 生成 `src/typings/i18n.d.ts` 类型文件，包含：
   - `I18nKeys` namespace -- 每个 namespace 下的 key 联合类型
   - `I18nFullKey` -- 完整的 `namespace.key` 联合类型
   - `I18n.$T` -- 类型安全的 `$t` 函数签名
4. HMR 模式下，JSON 文件变化时自动重新生成（基于 SHA-256 哈希比对）

### 效果

在代码中使用 `$t()` 时，TypeScript 会自动提示所有可用的 key：

```ts
$t("route.home") // 有类型提示和检查
$t("page.login.common.xx") // 不存在的 key 会报类型错误
```

## 如何新增翻译

### 步骤 1：添加 JSON 内容

在对应的 namespace JSON 文件中添加翻译文本：

```json
// zh-CN/page.json
{
  "myModule": {
    "title": "我的模块",
    "description": "这是我的模块描述"
  }
}
```

### 步骤 2：同步其他语言

在 `en-US/page.json` 中添加相同结构的英文翻译：

```json
{
  "myModule": {
    "title": "My Module",
    "description": "This is my module description"
  }
}
```

### 步骤 3：使用

类型文件会自动重新生成，直接在代码中使用：

```vue
<template>
  <h1>{{ $t("page.myModule.title") }}</h1>
</template>
```

## IDE 集成（i18n-ally）

项目配置了 [i18n Ally](https://marketplace.visualstudio.com/items?itemName=Lokalise.i18n-ally) VSCode 插件，提供以下开发体验：

- 在编辑器中直接显示翻译文本的内联预览
- 翻译缺失检测和提示
- 支持快速跳转到对应的语言文件

### 配置项

在 `.vscode/settings.json` 中：

```json
{
  "i18n-ally.displayLanguage": "zh-CN",
  "i18n-ally.enabledParsers": ["json"],
  "i18n-ally.enabledFrameworks": ["vue", "custom"],
  "i18n-ally.namespace": true,
  "i18n-ally.keystyle": "nested",
  "i18n-ally.localesPaths": ["apps/frontend/src/locales/langs"]
}
```

## dayjs 和 Naive UI 语言联动

语言切换时，App Store 会联动更新 dayjs 和 Naive UI 的语言设置：

### dayjs

通过 `setDayjsLocale()` 设置 dayjs 的语言，映射关系：

| 应用语言 | dayjs locale |
| -------- | ------------ |
| `zh-CN`  | `zh-cn`      |
| `en-US`  | `en`         |

### Naive UI

Naive UI 组件的语言通过 `NConfigProvider` 的 `locale` 和 `dateLocale` 属性控制，跟随应用语言切换自动更新。
