# 国际化

本项目后端实现了完整的国际化（i18n）支持，包括多语言翻译文件管理、运行时语言切换、类型安全的翻译键，以及开发阶段的热重载。

## 支持的语言

| 语言代码 | 语言     | 说明     |
| -------- | -------- | -------- |
| `zh-CN`  | 简体中文 | 默认语言 |
| `en-US`  | English  | 英文     |

默认语言通过环境变量 `DEFAULT_LANGUAGE` 配置，默认值为 `zh-CN`。

## 翻译文件结构

翻译文件位于 `src/locales/langs/` 目录，按语言和命名空间组织：

```
src/locales/langs/
  zh-CN/
    common.json
    auth.json
  en-US/
    common.json
    auth.json
```

每个 JSON 文件是一个命名空间，支持嵌套结构。例如 `common.json`：

```json
{
  "response": {
    "success": "请求成功",
    "badRequest": "请求格式错误",
    "validationError": "请求参数验证失败"
  }
}
```

## 加载机制

启动时，`load_and_flatten_locales` 函数遍历 `langs/` 下所有语言目录，读取每个 JSON 文件并将嵌套结构展平为点号分隔的键：

```python
def flatten_json(prefix: str, data: dict[str, str]) -> dict[str, str]:
    result: dict[str, str] = {}
    for key, value in data.items():
        new_key = f"{prefix}.{key}" if prefix else key
        if isinstance(value, dict):
            result.update(flatten_json(new_key, value))
        else:
            result[new_key] = value
    return result
```

展平后的结果形如：

```python
{
    "zh-CN": {
        "common.response.success": "请求成功",
        "common.response.badRequest": "请求格式错误",
        "auth.error.invalidCredentials": "用户名或密码错误",
    },
    "en-US": {
        "common.response.success": "Successful",
        # ...
    }
}
```

## I18n 类与 t() 函数

`I18n` 类是国际化的核心，提供翻译查找功能：

```python
class I18n:
    def __init__(self) -> None:
        locale_path = Path(__file__).parent.joinpath("langs")
        self.locales = load_and_flatten_locales(locale_path)

    @property
    def current_language(self) -> str:
        try:
            return ctx.language
        except (AttributeError, LookupError, ContextDoesNotExistError):
            return settings.DEFAULT_LANGUAGE

    def t(self, key: I18nKey, language: str | None = None, **kwargs) -> str:
        data = self.locales.get(language or self.current_language, {})
        return data.get(key, key).format(**kwargs)
```

全局导出的 `t()` 函数是 `i18n.t` 的别名，在代码中直接调用即可：

```python
from src.locales.i18n import t

message = t("common.response.success")  # -> "请求成功"
```

:::info
`current_language` 属性从请求上下文中读取当前语言设置。如果不在请求上下文中（如 Celery 任务），则回退到 `DEFAULT_LANGUAGE`。
:::

## I18nMiddleware

中间件从请求头 `Accept-Language` 中解析客户端语言偏好，并设置到当前请求上下文：

```python
class I18nMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, callback):
        language = resolve_language(request.headers.get("accept-language"), languages)
        if language and i18n.current_language != language:
            i18n.current_language = language
        response = await callback(request)
        return response
```

`resolve_language` 函数解析 `Accept-Language` 头的完整格式（包括 `q` 权重值），按权重降序匹配支持的语言列表，支持基础语言标签的模糊匹配（如 `zh` 匹配 `zh-CN`）。

## 类型安全

`src/locales/types.py` 文件由工具自动生成，包含所有合法翻译键的 `Literal` 类型定义：

```python
# Auto-generated by gen-i18n-types. Do not edit manually.
from typing import Literal, TypeAlias

LANGUAGE_TYPE: TypeAlias = Literal["zh-CN", "en-US"]
languages: set[LANGUAGE_TYPE] = {"zh-CN", "en-US"}

I18nKey = Literal[
    "auth.error.invalidCredentials",
    "common.response.alreadyExists",
    "common.response.authenticationFailed",
    # ...
]
```

`t()` 函数的参数类型为 `I18nKey`，IDE 能够提供自动补全，编译期即可发现拼写错误。

## 开发热重载

开发环境下，`watch_locale_files` 使用 `watchfiles` 库监听 `langs/` 目录的文件变化。当翻译文件被修改时，自动重新加载翻译数据并重新生成 `types.py` 类型文件：

```python
async def watch_locale_files():
    base_path = os.path.dirname(__file__)
    lang_path = os.path.join(base_path, "langs")
    types_path = os.path.join(base_path, "types.py")

    async for changes in awatch(lang_path):
        locales_data = load_and_flatten_locales(lang_path)
        namespaces = load_language_namespace(lang_path)
        # 重新生成 types.py ...
        write_py_file(types_path, generated_code)
```

:::tip
修改翻译文件后无需重启服务，`watchfiles` 会自动检测变化并更新内存中的翻译数据和类型定义文件，提升开发体验。
:::
