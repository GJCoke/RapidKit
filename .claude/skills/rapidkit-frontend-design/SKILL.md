This skill guides creation of distinctive, production-grade frontend interfaces for the **RapidKit** project.
All generated code MUST follow the project's tech stack, conventions, and styling patterns described below.

## Project Architecture

RapidKit is a **pnpm + Turborepo** monorepo with **uv workspace** for Python packages:

```
rapidkit/
├── apps/
│   ├── frontend/          # Vue 3 SPA (main target of this skill)
│   ├── backend/           # Python FastAPI backend
│   ├── desktop/           # Electron desktop wrapper
│   └── website/           # Documentation site
├── packages/
│   ├── axios/             # HTTP client wrapper (@rapidkit/axios)
│   ├── alova/             # Alova HTTP client (@rapidkit/alova)
│   ├── hooks/             # Shared Vue composables (@rapidkit/hooks)
│   ├── utils/             # Shared utilities (@rapidkit/utils)
│   ├── color/             # Color palette utilities (@rapidkit/color)
│   ├── editor/            # Editor component (@rapidkit/editor)
│   ├── builder/           # Build tooling (@rapidkit/builder)
│   ├── cli/               # CLI tool (@rapidkit/cli)
│   ├── common/            # Python shared package (rapidkit-common)
│   └── core/              # Python core package (rapidkit-core)
```

## Component Authoring Patterns

### Standard SFC Template

```vue
<script setup lang="ts">
  // Use lang="tsx" ONLY when inline JSX rendering is needed (e.g., data table column renderers)
  import { $t } from "@/locales"

  defineOptions({ name: "ModuleName" })

  // Type-only props (no runtime validation)
  const props = defineProps<{
    data: SomeType
  }>()

  // Typed emits
  const emit = defineEmits<{
    change: [value: string]
  }>()

  // Two-way binding
  const model = defineModel<string>()
</script>

<template>
  <!-- Template content using UnoCSS utility classes + NaiveUI components -->
</template>

<style scoped>
  /* Only when utility classes are insufficient */
</style>
```

### Key Conventions

- **File names**: kebab-case (`user-operate-drawer.vue`)
- **Component names**: PascalCase via `defineOptions({ name: "UserOperateDrawer" })`
- **NaiveUI components**: Auto-imported, use directly (`<NCard>`, `<NDataTable>`, `<NGrid>`, `<NButton>`, etc.)
- **Icons**: Use `<SvgIcon icon="carbon:icon-name" />` component or auto-imported icon components with `icon-` prefix
- **Path aliases**: `@` = `src/`, `~` = project root
- **View structure**: `index.vue` + `modules/` directory + optional `composables/` directory

### NaiveUI Component Usage

NaiveUI components are **auto-imported** via `NaiveUiResolver` — never manually import them.

Commonly used components:

- **Layout**: `NCard`, `NGrid`/`NGi`, `NSpace`, `NScrollbar`, `NLayout`/`NLayoutSider`/`NLayoutContent`
- **Data Display**: `NDataTable`, `NTag`, `NTimeline`/`NTimelineItem`, `NStatistic`, `NDescriptions`, `NEmpty`
- **Form**: `NForm`/`NFormItem`, `NInput`, `NSelect`, `NDatePicker`, `NSwitch`, `NCheckbox`, `NRadio`
- **Feedback**: `NModal`, `NDrawer`, `NPopconfirm`, `NSpin`, `NProgress`
- **Navigation**: `NMenu`, `NTabs`/`NTabPane`, `NBreadcrumb`, `NButtonGroup`/`NButton`

**Card pattern** — all cards use the `card-wrapper` shortcut:

```vue
<NCard :bordered="false" class="card-wrapper">
  <!-- content -->
</NCard>
```

**Grid layout pattern**:

```vue
<NGrid :x-gap="16" :y-gap="16" responsive="screen" item-responsive>
  <NGi span="24 s:24 m:8"><!-- column content --></NGi>
  <NGi span="24 s:24 m:16"><!-- column content --></NGi>
</NGrid>
```

**Theme CSS variables** available from NaiveUI (use via bracket syntax):

- `var(--primary-color)`, `var(--success-color)`, `var(--warning-color)`, `var(--error-color)`
- `var(--text-color-1)`, `var(--text-color-2)`, `var(--text-color-3)`
- `var(--border-color)`, `var(--divider-color)`
- `var(--n-color-modal)`, `var(--n-color-target)` (NaiveUI internal)

## UnoCSS Styling System

### Configuration

UnoCSS is configured with:

- **`presetWind3`** with `dark: "class"` — full Tailwind CSS compatible utility classes
- **`@unocss/transformer-directives`** — enables `@apply` and `@screen` in `<style>` blocks
- **`@unocss/transformer-variant-group`** — enables `hover:(bg-red text-white)` syntax

### Project Shortcuts

**Global shortcut:**

- `card-wrapper` = `rd-8px shadow-sm`

**Flex shortcuts:**

- `flex-center` = `flex justify-center items-center`
- `flex-x-center` = `flex justify-center`
- `flex-y-center` = `flex items-center`
- `flex-col` = `flex flex-col`
- `flex-col-center` = `flex-center flex-col`
- `flex-col-stretch` = `flex-col items-stretch`
- `i-flex-center`, `i-flex-x-center`, `i-flex-y-center` (inline-flex variants)
- `i-flex-col`, `i-flex-col-center`, `i-flex-col-stretch` (inline-flex column variants)
- `flex-1-hidden` = `flex-1 overflow-hidden`

**Position shortcuts:**

- `absolute-lt`, `absolute-rt`, `absolute-lb`, `absolute-rb` (corner positioning)
- `absolute-tl`, `absolute-tr`, `absolute-bl`, `absolute-br` (corner aliases)
- `absolute-center` = `absolute left-0 top-0 flex-center size-full`
- `fixed-lt`, `fixed-rt`, `fixed-lb`, `fixed-rb`, `fixed-center`
- `fixed-tl`, `fixed-tr`, `fixed-bl`, `fixed-br` (fixed corner aliases)

**Text shortcuts:**

- `nowrap-hidden` = `overflow-hidden whitespace-nowrap`
- `ellipsis-text` = `nowrap-hidden text-ellipsis`

### Theme Colors

UnoCSS theme colors are mapped to CSS custom properties:

```
primary, info, success, warning, error
```

Each with palette shades: `50, 100, 200, 300, 400, 500, 600, 700, 800, 900, 950`

Usage: `text-primary`, `bg-success-100`, `border-error-500`

Additional theme colors: `nprogress`, `container`, `layout`, `inverted`, `base-text`

**Default color values:**

- Primary: `#4361EE`
- Info: `#2080f0`
- Success: `#2EC4B6`
- Warning: `#F4A62A`
- Error: `#EF476F`

### Layout Dimensions

Default layout token values from `theme/settings.ts`:

- Header height: `56px`
- Tab height: `44px`
- Sider width: `220px`
- Sider collapsed width: `64px`
- Default layout mode: `vertical`
- Default scroll mode: `content`

### Box Shadow Tokens

Available via `theme/vars.ts`:

- `shadow-header`, `shadow-sider`, `shadow-tab`

### Icon Font Sizes

- `text-icon-xs` (0.875rem), `text-icon-small` (1rem), `text-icon` (1.125rem), `text-icon-large` (1.5rem), `text-icon-xl` (2rem)

### Styling Priorities

Follow this order of preference:

1. **UnoCSS utility classes** (primary method) — use for all standard styling
2. **UnoCSS shortcuts** — use project-defined shortcuts when applicable
3. **Bracket syntax** for CSS variables — `text-[var(--primary-color)]`, `bg-[var(--n-color-modal)]`
4. **Arbitrary values** — `w-220px`, `gap-16px`, `text-13px`
5. **Scoped `<style>`** — only when utilities are insufficient (complex animations, `:deep()` selectors for NaiveUI internal elements)
6. **`@apply` directive** — group repeated utility patterns in `<style scoped>` when needed

### Common Patterns

```vue
<!-- Flex layout with gap -->
<div class="flex-col-stretch gap-16px">

<!-- Card with icon title -->
<div class="flex items-center gap-8px text-15px font-600 mb-12px">
  <SvgIcon icon="carbon:activity" class="text-16px text-[var(--primary-color)]" />
  {{ $t("page.module.title") }}
</div>

<!-- Hover effect row -->
<div class="flex items-center justify-between px-12px py-8px rd-8px transition-colors duration-200 hover:bg-[var(--n-color-modal)]">

<!-- Metric badge -->
<div class="px-10px py-6px rd-8px bg-[var(--n-color-modal)]">
  <span class="text-20px font-700 tabular-nums">{{ value }}</span>
</div>

<!-- Responsive grid -->
<div class="grid grid-cols-3 gap-10px lg:grid-cols-6">

<!-- Variant group syntax -->
<div class="hover:(bg-primary text-white shadow-lg)">

<!-- Deep selector for NaiveUI internals -->
<style scoped>
  .wrapper :deep(.n-card__content) {
    flex: 1;
    display: flex;
    flex-direction: column;
  }
</style>
```

## Design Thinking

Before coding, understand the context and commit to an aesthetic direction that fits the **admin dashboard** context:

- **Purpose**: What problem does this interface solve? Who uses it (admins, operators, developers)?
- **Tone**: RapidKit is a modern admin panel — favor clean, professional, data-dense aesthetics. Prioritize readability and usability over decoration.
- **Consistency**: Follow existing visual patterns — card-based layouts, subtle hover states, tabular-nums for data, CSS variable-driven theming.
- **Differentiation**: Add personality through micro-interactions, thoughtful spacing, smart data visualization, and polished details.

Implement working code that is:

- **Production-grade** and functional within the existing tech stack
- **Visually cohesive** with the existing dashboard aesthetic
- **Responsive** using NaiveUI's responsive system and UnoCSS responsive prefixes
- **Theme-aware** using CSS variables for colors so light/dark mode works automatically

## Frontend Aesthetics Guidelines

Focus on:

- **Typography**: Use the project's font stack. Differentiate hierarchy with `font-600`/`font-700` weights, `text-13px`/`text-15px`/`text-20px` sizes, and `text-[var(--text-color-N)]` color variants.
- **Color & Theme**: Always use CSS variables (`var(--primary-color)`, theme palette colors). Never hardcode colors that should change with theme. Accent colors for status: green=success, orange=warning, red=error.
- **Motion**: Subtle transitions on interactive elements (`transition-colors duration-200`). Pulse animations for live status indicators. Chart value animations via ECharts `valueAnimation`.
- **Spatial Composition**: Card-based layouts with consistent `gap-16px`. NGrid for responsive multi-column layouts. Generous padding inside cards (`px-12px py-8px`).
- **Data Density**: Admin dashboards should be information-rich. Use `tabular-nums` for numeric data. Compact but readable list items. Smart use of tags, badges, and status indicators.

## Rules

### Frontend Rules

- NEVER import NaiveUI components manually — they are auto-imported
- NEVER use raw CSS for styling that UnoCSS utilities can handle
- NEVER hardcode colors — use CSS variables or UnoCSS theme colors
- ALWAYS use `$t()` for user-facing text (i18n)
- ALWAYS add `defineOptions({ name: "ComponentName" })` to every component
- ALWAYS use `card-wrapper` class on `NCard` components
- ALWAYS use `:bordered="false"` on cards (project convention)
- PREFER UnoCSS arbitrary values (`text-13px`, `gap-16px`) over Tailwind named sizes for pixel-precise control
- PREFER `<SvgIcon icon="carbon:xxx" />` for icons
- PREFER `flex-col-stretch`, `flex-center`, and other project shortcuts over raw flex utilities

### Git Rules

- NEVER run `git add` or `git commit` — leave all version control operations to the user
- NEVER stage or commit files automatically after generating code
